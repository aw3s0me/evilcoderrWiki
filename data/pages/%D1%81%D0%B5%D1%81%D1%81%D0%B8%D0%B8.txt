====== Сессии ======
  * Сессии - механизм для идентификации браузера и для создания файла в котором хранятся переменные сеанса для него
  * для сохранения состояния между переходами
  * То есть если юзер перешел на другую страницу, то хттп протокол не позволяет узнать тот же человек или нет
===== Создание =====
Технология создана, потому что сервер не держит ПОСТОЯННУЮ связь с клиентом и каждый раз создается новый запрос и НЕТ связи с предыдущим
===== Как работают =====
  * идентификация браузера [даем ему уник id и теперь он его всегда юзает
Идентификатор - это обычная переменная. По умолчанию ее имя - PHPSESSID. 
==== Запросы ====
  * для связи с браузером отправляет запрос с id 2мя способами:
  - Cookie или [Привязать к IP надо]
  - URL [НЕ БЕЗОПАСНО]
=== Способ настройки ===
  - (php.ini)
  - ini_set()
  - в файлах настроек сервера
==== Cookies ====
  - при каждом вызове session_start() клиенту устанавливается кука
  - браузер при запросе возвращает эту куку и имеет id
==== URL ====
  * кука не ставится
  * дописывает к каждой ссылке: <a href="/index.php?PHPSESSID=4823905823905230efuerifg4">Index</a>
  * к формам добавляет скрытое поле <input type="hidden"...>
===== Привязка файла =====
Происходит через массив $_SESSION
    $_SESSION['test']="Hello world!"; 
  * В этом массиве хранятся переменные которые доступны в различных скриптах
  * Сам занимается сборкой мусора
===== Зачем юзать? =====
  * Применяется когда нужно пользователю, а не для того чтобы делать препятствия (в любой момент избавиться от ID) ДЛЯ ОГРАНИЧЕНИЯ ЗАПРОСА сессия не годится
  * Сессия - СЕАНС РАБОТЫ С САЙТОМ, хочешь поработать - новая сессия
Если надо запомнить пользователя, то ему ставится кука, с его ID. При след входе опознать по куке
===== Работа с сессией =====
  * Открытие сессии - session_start()
  * Удаление переменных - unset($_SESSION['username']); при register_globals = off
  * Удаление переменных - session_unregister('var'); при register_globals = on
  * Регистрация переменных - $_SESSION['name'] = "value";
===== Отличия кук от сессий =====
Почти тоже самое, только сессии хранятся в файле, куки в браузере
  * Время жизни сессии - 15 минут
  * Считывает ID из сессии
  * Нельзя юзать чистые куки, небезопасно, лучше сессии
