===== Отличия Pipe от Fifo =====
  * Для взаимодействия Fifo использует специальный файл с расширением .fifo, который не имеет физического размера и не хранится в файловой системе
  * Pipe Хранится в адресном пространстве
  * Все после считывания из pipe удаляется, то есть pipe повторная инф-ия не может быть прочитана повторно
  * Если FIFO открыт для чтения, то блокируется пока не запишутся данные
  * чтение блокирует pipe пока данные не досутпны
  * The kernel maintains exactly one pipe object for each FIFO special file that is opened by at least one process.
  * Opening a FIFO for reading normally blocks until some other process opens the same FIFO for writing, and vice versa.
  * проверка на выход, когда размер считывания равен 0, то break, поток пуст
  * когда у пайпа доходит до конца размер, то есть он забит, пайп блокируется\но ошибка не выводится
  * У пайпа нет имени, и фифо - имя == файл
  * Взаимодействие у пайпа только через родственные процессы
